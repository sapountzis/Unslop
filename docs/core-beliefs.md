# Core Beliefs

1. Repository-local knowledge is the only reliable context for agents.
2. `AGENTS.md` is a map, not an encyclopedia.
3. `make check` is the canonical quality gate.
4. Architecture constraints must be mechanically enforced.
5. Boundary validation is mandatory for external inputs.
6. Technical debt is paid continuously in small increments.
7. Prefer boring, composable tools with predictable behavior.
8. Keep docs fresh and cross-linked so future agents can reason correctly.
