# Quality Documentation

Use this directory to track quality posture and technical debt.

## Files
- `docs/quality/QUALITY_SCORE.md`: current quality scores by domain and layer.
- `docs/quality/tech-debt.md`: actionable debt backlog tied to quality gaps.

## Operating Loop
- Follow `docs/runbooks/quality-review.md` for scoring/debt updates.
- Follow `docs/runbooks/docs-freshness.md` to keep docs synchronized with changes.
- Keep weakest quality areas linked to concrete debt items.
